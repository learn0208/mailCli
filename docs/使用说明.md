# mailCli 使用说明

**mailCli** 是跨协议邮件 CLI，支持 **Microsoft Exchange EWS** 与 **IMAP/SMTP**（Gmail、企业邮箱等）。在指定文件夹中 **查询邮件**、**发送邮件**、按 **ItemId/UID 查看单封邮件** 正文与头信息。

英文概览见 [README.md](../README.md)；**功能清单与计划**见 [ROADMAP.md](../ROADMAP.md)；预编译包见 [GitHub Releases](https://github.com/learn0208/mailCli/releases)。

## 1. 准备与配置

### 1.1 编译

在项目根目录执行：

```bash
go build -o mailcli.exe ./cmd/mailcli
```

（Linux/macOS 可将输出文件名改为 `mailcli`。）

### 1.2 配置文件（YAML）

默认依次查找：**当前目录** `.mailcli.yaml` → `~/.mailcli.yaml` → `~/.ews-cli.yaml`。也可用 **`--config`** 指定路径。配置文件需为 **UTF-8** 编码。**勿将授权码提交到 Git**（用 `MAILCLI_PASSWORD`）。

每个 profile 可设置 `protocol: ews`（默认，可省略）或 `protocol: imap`（IMAP 查信 + SMTP 发信）。

#### IMAP / SMTP profile 示例（最简）

指定 `provider` 或 `user@常见域名` 后，**无需**手写 `imap` / `smtp`，会自动填入主机与 TLS：

```yaml
profiles:
  qq:
    protocol: imap
    provider: qq
    user: 123456789@qq.com
```

查看合并后的实际连接参数：`mailcli profile show`

| 字段 | 说明 |
|------|------|
| `provider` | 预设 id：`qq`、`163`、`gmail` 等（见 `mailcli providers list`） |
| `imap.host` / `smtp.host` | 可选；省略时由 provider 填充 |
| `imap.tls` / `smtp.tls` / `smtp.starttls` | 可选；省略时按 provider 与端口默认 |

环境变量：`MAILCLI_IMAP_HOST`、`MAILCLI_SMTP_HOST`、`MAILCLI_PROVIDER`（如 `qq`、`163`、`gmail`）。`search`/`show` 的 `item_id` 为文件夹内 **UID**；`show` 的 `--folder` 须与搜索时一致。

**常见邮箱配置文档**（建议收藏，便于日后配置）：

| 资源 | 说明 |
|------|------|
| [docs/providers/README.md](providers/README.md) | 各服务商分步指南（QQ/163/Gmail…） |
| [docs/examples/providers/](examples/providers/) | 可直接复制的 `~/.mailcli.yaml` 片段 |
| [docs/providers.md](providers.md) | 文档索引 |

```bash
mailcli providers list
mailcli providers doc qq          # 终端输出 QQ 邮箱完整配置说明
mailcli providers show qq --user 123456789@qq.com
mailcli discover --user you@163.com
```

只需 `user: you@qq.com` 或 `provider: qq`，主机地址会自动填入。

#### QQ 邮箱能否扫码登录？

**不能。** IMAP/SMTP 是标准协议，只支持用户名 + 密码（对 QQ/163 而言是 **授权码/客户端密码**，不是网页登录密码）。QQ 邮箱 App/网页的 **扫码登录** 属于腾讯 Web 会话，第三方 CLI **无法** 使用。

QQ 邮箱正确做法：在 [mail.qq.com](https://mail.qq.com) → 设置 → 账户 → 开启 IMAP/SMTP → 生成 **授权码**，然后 `export MAILCLI_PASSWORD='授权码'`。

**顶层（全局）**

| 字段 | 说明 |
|------|------|
| `user_agent` | 可选。HTTP `User-Agent`，未设置时默认为 `mailcli/<版本>`（产品名 mailCli）。 |

**`profiles.<名称>`（账号）**

| 字段 | 说明 |
|------|------|
| `endpoint` | EWS 地址，须为 **HTTPS**，例如 `https://mail.example.com/EWS/Exchange.asmx`。 |
| `user` | 登录用户：可为邮箱 `user@domain.com`，或短账号（见 `smtp_address`）。 |
| `auth_type` | `basic`（默认）、`ntlm`、`oauth`。 |
| `domain` | NTLM 时常用：与 `user` 组成 `DOMAIN\user`；若 `user` 无 `@`，也会用于推断默认发件地址 `user@domain`。 |
| `password` | 可选；更推荐 **`MAILCLI_PASSWORD`**（或旧名 `EWS_PASSWORD`）。 |
| `access_token` | OAuth 时使用（或 `MAILCLI_TOKEN` / `EWS_TOKEN`）。 |
| `smtp_address` | 可选。当 `user` 不是完整邮箱时，作为 **`send` 默认 `--from`** 的 SMTP 地址。 |
| `display_name` | 可选。UTF-8 发件人显示名（与 `--from-name` 二选一优先级见下文）。 |
| `from_display_name_plain_utf8` | 可选。`true` 时不对中文显示名做 RFC2047 包装（一般保持默认 `false` 即可）。 |
| `from_address_only` | 可选。`true` 时用 **纯 ASCII 邮箱** 作为发件显示名，减轻部分环境下 GAL 昵称乱码。 |

**示例结构（请勿把真实密码提交到版本库）：**

```yaml
user_agent: "MyScript/1.0"
profiles:
  default:
    endpoint: https://mail.example.com/EWS/Exchange.asmx
    user: zhang.san
    auth_type: ntlm
    domain: corp.example.com
    smtp_address: zhang.san@mail.example.com
```

### 1.3 环境变量（覆盖配置文件中的同名项）

优先使用 **`MAILCLI_*`**；**`EWS_*`** 仍兼容。

| 变量 | 作用 |
|------|------|
| `MAILCLI_ENDPOINT` / `EWS_ENDPOINT` | 覆盖 `endpoint` |
| `MAILCLI_USER` / `EWS_USER` | 覆盖 `user` |
| `MAILCLI_PASSWORD` / `EWS_PASSWORD` | 覆盖 `password` |
| `MAILCLI_TOKEN` / `EWS_TOKEN` | OAuth 访问令牌 |
| `MAILCLI_AUTH_TYPE` / `EWS_AUTH_TYPE` | 覆盖 `auth_type` |
| `MAILCLI_DOMAIN` / `EWS_DOMAIN` | 覆盖 `domain` |
| `MAILCLI_SMTP_ADDRESS` / `EWS_SMTP_ADDRESS` | 覆盖 `smtp_address` |
| `MAILCLI_USER_AGENT` / `EWS_USER_AGENT` | 覆盖 `user_agent` |
| `MAILCLI_TIMEOUT` / `EWS_TIMEOUT` | HTTP 超时秒数（默认 30） |
| `MAILCLI_SERVER_VERSION` / `EWS_SERVER_VERSION` | Exchange 版本（默认 `Exchange2016`） |
| `MAILCLI_IMAP_HOST` | 覆盖 `imap.host` |
| `MAILCLI_SMTP_HOST` | 覆盖 `smtp.host` |

### 1.4 全局命令行参数（所有子命令可用）

| 参数 | 说明 |
|------|------|
| `--config <路径>` | 配置文件路径（支持 `~`）。 |
| `--profile <名称>` | 使用的 profile，默认 `default`。 |
| `--verbose` | 打印原始 HTTP 请求/响应（排错用）。 |
| `--endpoint` | 临时覆盖 EWS URL。 |
| `--user` | 临时覆盖用户名/邮箱。 |
| `-p` / `--password` | 临时密码（更推荐 `MAILCLI_PASSWORD`）。 |
| `--auth-type` | `basic` \| `ntlm` \| `oauth`。 |
| `--user-agent` | 临时覆盖 HTTP User-Agent。 |

## 2. 查询邮件（`search`）

结果按 **接收时间降序** 输出；默认时间窗 **最近 30 天**（EWS）或关键词搜索时扫描邮箱内最近一批邮件（IMAP，见下），默认最多 **10 条**。

| 协议 | 机制 | `ITEM_ID` 含义 |
|------|------|----------------|
| **EWS** | 服务端 FindItem | EWS ItemId |
| **IMAP** | UidSearch + 本地过滤 | 文件夹内 **UID** |

### 2.1 常用示例（通用）

```bash
# 主题包含「周报」
mailcli search --subject "周报"

# 关键词：匹配主题、发件人、正文（IMAP 推荐）
mailcli search "招商银行"
mailcli search --query "信用管家"

# JSON 输出（脚本）
mailcli search --subject "周报" --output json

# 指定时间（传了 --since/--until 后 EWS 用该范围；IMAP 关键词搜索也会套用）
mailcli search --since "2026-05-01" --until "2026-05-15" --from "boss@"

# 仅未读、带附件、已发送文件夹
mailcli search --folder sent --unread --attachments --limit 20
```

**QQ 邮箱示例：**

```bash
export MAILCLI_PASSWORD='授权码'
mailcli search --subject "招商银行信用卡"
mailcli search "信用管家" --limit 5
mailcli --verbose search "招商银行"   # 排错时看 stderr
```

### 2.2 主要参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--subject` | 空 | 主题包含（忽略大小写）。 |
| `--query` | 空 | **IMAP**：主题 + 发件人 + 正文任意包含；可用位置参数 `mailcli search "词"`。 |
| `--body` | 空 | 正文包含（EWS 服务端；IMAP 拉取正文后本地匹配）。 |
| `--from` | 空 | 发件人包含（EWS：Sender；IMAP：信封发件人）。 |
| `--to` | 空 | 收件人包含。 |
| `--since` / `--until` | 见下 | 时间范围；支持 RFC3339、日期、`7 days ago` 等。 |
| `--folder` | `Inbox` | 文件夹名（IMAP 须与 `show` 一致）。 |
| `--unread` / `--read` | false | 仅未读 / 仅已（勿同时）。 |
| `--attachments` | false | 仅含附件。 |
| `--limit` | `10` | 最多返回条数。 |
| `--default-days` | `30` | **EWS**：未传时间时默认最近 N 天。 |
| `--no-default-date-window` | false | 不传时间时不加默认 N 天限制。 |
| `--output` | `table` | `table` 或 `json`。 |

**IMAP 说明：**

- 中文关键词**不能**作为服务器 SEARCH 条件，只在本地匹配；发件人显示名可能在正文里才有，建议用 `--query`。
- 带 `--query` / `--subject` 等时，默认在邮箱中取 **最新约 800 封** 再过滤（`--verbose` 可见 UID 数）；要更全可加 `--no-default-date-window`（更慢）。
- 表格 **ITEM_ID** = UID，供 `show` 使用。

### 2.3 辅助命令

```bash
mailcli discover --user someone@example.com   # EWS/IMAP 主机提示
mailcli providers list
mailcli providers doc qq
mailcli profile show                          # 查看合并后的连接参数
```

## 3. 发送邮件（`send`）

| 协议 | 机制 |
|------|------|
| **EWS** | CreateItem（SendAndSaveCopy），可选复核「已发送」 |
| **IMAP** | **SMTP** 发信，可选经 IMAP 复核「已发送」 |

### 3.1 必填与正文

| 参数 | 说明 |
|------|------|
| `--to` | 必填。收件人，逗号或分号分隔。 |
| `--subject` | 必填。 |
| `--text` | 与 `--html` **二选一**：纯文本正文。 |
| `--html` | HTML 正文。 |
| `--cc` / `--bcc` | 抄送 / 密送。 |
| `--attach` | 附件路径，可重复多次。 |
| `--importance` | `high` \| `normal` \| `low`（**仅 EWS**）。 |
| `--from` | 发件 SMTP；省略时按 **`smtp_address`**，否则 **`user@domain`**（`user` 无 `@` 时用 `domain` 拼接）。 |
| `--from-name` | 覆盖配置里的 `display_name`；中文等非 ASCII 默认会做 RFC2047 UTF-8 编码。 |
| `--from-name-plain-utf8` | 不对显示名做 RFC2047（少数服务器需试）。 |
| `--from-address-only` | 用纯 ASCII 邮箱作为显示名，减轻部分环境发件人乱码；建议配合完整 `--from`。 |

### 3.2 发送后复核「已发送」（默认开启）

| 参数 | 默认 | 说明 |
|------|------|------|
| `--verify-sent-wait` | `5` | 发送成功后等待的秒数，再在「已发送」中按主题查找副本（0～120）。 |
| `--no-verify-sent` | false | 不等待、不复核（脚本/CI 可加）。 |
| `--json` | false | 输出 JSON，含 `sent_verified`、`sent_verify_item_id` 等字段。 |

成功时典型输出：

- `发送成功。`
- 若 EWS 返回 ItemId 会多一行；
- 若复核命中：`已在「已发送」确认副本：… item_id=…`
- 若未命中：会提示可能延迟，请到网页「已发送」再确认。

### 3.3 示例

**EWS：**

```bash
mailcli send --from zhang.san@mail.example.com \
  --to "a@b.com,c@d.com" --subject "通知" --text "正文内容"
```

**QQ / IMAP（SMTP）：**

```bash
export MAILCLI_PASSWORD='授权码'
mailcli send --to 123456789@qq.com --subject "mailcli 测试" --text "这是一封测试邮件。"
mailcli send --to other@example.com --subject "通知" --text "正文" --json
mailcli send --to x@y.z --subject "无复核" --text "hi" --no-verify-sent
```

## 4. 查看单封邮件（`show`）

根据 **`search` 输出的 ITEM_ID** 读取单封：信头 + 正文。附件仅元数据（名称/大小），不下载二进制。

| 协议 | `ITEM_ID` | 说明 |
|------|-----------|------|
| **EWS** | EWS ItemId | 可选 `--change-key` |
| **IMAP** | 数字 **UID** | **`--folder` 须与 search 时一致**（默认 Inbox） |

| 参数 | 说明 |
|------|------|
| `--item-id` | **必填**。EWS ItemId 或 IMAP UID。 |
| `--change-key` | 可选（**EWS**）。 |
| `--folder` | **IMAP** 文件夹，默认 `Inbox`。 |
| `--format` | `text`（默认）、`html`、`json`。 |

```bash
# EWS
mailcli show --item-id "AAMkAG..." --format json

# IMAP（先 search 得到 UID）
mailcli search --subject "招商银行信用卡"
mailcli show --item-id 1460 --format text
mailcli show --item-id 1485 --format html   # HTML 邮件
```

## 5. 排错提示

**通用**

- 密码/授权码用 **`MAILCLI_PASSWORD`**，不要写入 YAML 并提交 Git。
- 加 **`--verbose`**：EWS 打印 SOAP/HTTP；IMAP 打印 SEARCH/FETCH 进度（stderr）。
- `mailcli profile show` 确认 protocol、imap/smtp 主机是否正确。

**EWS**

- 认证失败：检查 `auth_type`、`domain`、密码。
- `search` 复杂条件可能 `ErrorInvalidRequest`，可只保留 `--subject`。
- `show` 的 ChangeKey 报错可省略 `--change-key`。

**IMAP / QQ**

- 登录失败：QQ 须用 **授权码**，不是 QQ 密码；确认已开启 IMAP/SMTP。
- `search` 无结果：发件人名在正文时用 `--query`；扩大范围 `--no-default-date-window`。
- `show` 报 UID not found：确认 `--folder` 与 search 一致；`item-id` 填 UID 数字。
- `fetched 0/N envelopes`：多为序号/UID 混用问题（当前版本已用 UidSearch+UidFetch）。

版本号：`mailcli --version`。
