# mailCli 使用说明

**mailCli** 是跨协议邮件 CLI（当前实现 **Microsoft Exchange EWS**；后续计划支持 IMAP/SMTP）。在指定文件夹中 **查询邮件**、**发送邮件**、按 **ItemId 查看单封邮件** 正文与头信息。

英文概览见 [README.md](../README.md)；**功能清单与计划**见 [ROADMAP.md](../ROADMAP.md)；预编译包见 [GitHub Releases](https://github.com/learn0208/mailCli/releases)。

---

## 1. 准备与配置

### 1.1 编译

在项目根目录执行：

```bash
go build -o mailcli.exe ./cmd/mailcli
```

（Linux/macOS 可将输出文件名改为 `mailcli`。）

### 1.2 配置文件（YAML）

默认读取 `~/.mailcli.yaml`；若不存在则回退 `~/.ews-cli.yaml`。也可用 **`--config`** 指定路径。配置文件需为 **UTF-8** 编码。

每个 profile 可设置 `protocol: ews`（默认，可省略）。

**顶层（全局）**

| 字段 | 说明 |
|------|------|
| `user_agent` | 可选。HTTP `User-Agent`，未设置时默认为 `mailcli/<版本>`（产品名 mailCli）。 |

**`profiles.<名称>`（账号）**

| 字段 | 说明 |
|------|------|
| `endpoint` | EWS 地址，须为 **HTTPS**，例如 `https://mail.example.com/EWS/Exchange.asmx`。 |
| `user` | 登录用户：可为邮箱 `user@domain.com`，或短账号（见 `smtp_address`）。 |
| `auth_type` | `basic`（默认）、`ntlm`、`oauth`。 |
| `domain` | NTLM 时常用：与 `user` 组成 `DOMAIN\user`；若 `user` 无 `@`，也会用于推断默认发件地址 `user@domain`。 |
| `password` | 可选；更推荐 **`MAILCLI_PASSWORD`**（或旧名 `EWS_PASSWORD`）。 |
| `access_token` | OAuth 时使用（或 `MAILCLI_TOKEN` / `EWS_TOKEN`）。 |
| `smtp_address` | 可选。当 `user` 不是完整邮箱时，作为 **`send` 默认 `--from`** 的 SMTP 地址。 |
| `display_name` | 可选。UTF-8 发件人显示名（与 `--from-name` 二选一优先级见下文）。 |
| `from_display_name_plain_utf8` | 可选。`true` 时不对中文显示名做 RFC2047 包装（一般保持默认 `false` 即可）。 |
| `from_address_only` | 可选。`true` 时用 **纯 ASCII 邮箱** 作为发件显示名，减轻部分环境下 GAL 昵称乱码。 |

**示例结构（请勿把真实密码提交到版本库）：**

```yaml
user_agent: "MyScript/1.0"
profiles:
  default:
    endpoint: https://mail.example.com/EWS/Exchange.asmx
    user: zhang.san
    auth_type: ntlm
    domain: corp.example.com
    smtp_address: zhang.san@mail.example.com
```

### 1.3 环境变量（覆盖配置文件中的同名项）

优先使用 **`MAILCLI_*`**；**`EWS_*`** 仍兼容。

| 变量 | 作用 |
|------|------|
| `MAILCLI_ENDPOINT` / `EWS_ENDPOINT` | 覆盖 `endpoint` |
| `MAILCLI_USER` / `EWS_USER` | 覆盖 `user` |
| `MAILCLI_PASSWORD` / `EWS_PASSWORD` | 覆盖 `password` |
| `MAILCLI_TOKEN` / `EWS_TOKEN` | OAuth 访问令牌 |
| `MAILCLI_AUTH_TYPE` / `EWS_AUTH_TYPE` | 覆盖 `auth_type` |
| `MAILCLI_DOMAIN` / `EWS_DOMAIN` | 覆盖 `domain` |
| `MAILCLI_SMTP_ADDRESS` / `EWS_SMTP_ADDRESS` | 覆盖 `smtp_address` |
| `MAILCLI_USER_AGENT` / `EWS_USER_AGENT` | 覆盖 `user_agent` |
| `MAILCLI_TIMEOUT` / `EWS_TIMEOUT` | HTTP 超时秒数（默认 30） |
| `MAILCLI_SERVER_VERSION` / `EWS_SERVER_VERSION` | Exchange 版本（默认 `Exchange2016`） |

### 1.4 全局命令行参数（所有子命令可用）

| 参数 | 说明 |
|------|------|
| `--config <路径>` | 配置文件路径（支持 `~`）。 |
| `--profile <名称>` | 使用的 profile，默认 `default`。 |
| `--verbose` | 打印原始 HTTP 请求/响应（排错用）。 |
| `--endpoint` | 临时覆盖 EWS URL。 |
| `--user` | 临时覆盖用户名/邮箱。 |
| `-p` / `--password` | 临时密码（更推荐 `MAILCLI_PASSWORD`）。 |
| `--auth-type` | `basic` \| `ntlm` \| `oauth`。 |
| `--user-agent` | 临时覆盖 HTTP User-Agent。 |

## 2. 查询邮件（`search`）

在指定文件夹中执行 **FindItem**（服务端过滤），结果按 **接收时间降序**，默认只查 **最近 7 天**、最多 **10 条**（可改）。

### 2.1 常用示例

```text
# 主题包含「周报」，默认最近 7 天、最多 10 条
mailcli search --config .\.mailcli.yml --subject "周报"

# JSON 输出（便于脚本）
mailcli search --config .\.mailcli.yml --subject "周报" --output json

# 指定时间范围（与「默认 7 天」二选一：只要传了 --since 或 --until 任一，即不再自动套 7 天）
mailcli search --config .\.mailcli.yml --since "2026-05-01" --until "2026-05-15" --from "boss@"

# 仅未读、仅带附件、在「已发送」中查
mailcli search --config .\.mailcli.yml --folder sent --unread --attachments --limit 20
```

### 2.2 主要参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--subject` | 空 | 主题包含（忽略大小写）。 |
| `--body` | 空 | 正文包含。 |
| `--from` | 空 | 发件人地址包含（按 **Sender** 字段）。 |
| `--to` | 空 | 收件人包含。 |
| `--since` / `--until` | 见下 | 接收时间范围；支持 RFC3339、日期、`7 days ago` 等（见 `internal/timeparse`）。 |
| `--folder` | `Inbox` | 文件夹：`inbox`、`sent`/`sentitems`、`drafts`、`deleted`/`deleteditems`、`junk` 等。 |
| `--unread` / `--read` | false | 仅未读 / 仅已读（勿同时开两个）。 |
| `--attachments` | false | 仅含附件。 |
| `--limit` | `10` | 最多返回条数。 |
| `--default-days` | `7` | 当 **未** 传 `--since` 且 **未** 传 `--until` 时，自动限制为最近 N 天。 |
| `--no-default-date-window` | false | 不传时间参数时 **不要** 自动加「最近 N 天」限制。 |
| `--output` | `table` | `table` 或 `json`。 |

表格列中的 **ITEM_ID** 用于下一节的 **`show`**。

### 2.3 辅助命令 `discover`

仅根据邮箱域名打印常见 EWS URL 提示（**不做网络自动发现**）：

```text
mailcli discover --user someone@example.com
```

## 3. 发送邮件（`send`）

通过 **CreateItem**（`SendAndSaveCopy`）发送，并保存副本到「已发送」。

### 3.1 必填与正文

| 参数 | 说明 |
|------|------|
| `--to` | 必填。收件人，逗号或分号分隔。 |
| `--subject` | 必填。 |
| `--text` | 与 `--html` **二选一**：纯文本正文。 |
| `--html` | HTML 正文。 |
| `--cc` / `--bcc` | 抄送 / 密送。 |
| `--attach` | 附件路径，可重复多次。 |
| `--importance` | `high` \| `normal` \| `low`。 |
| `--from` | 发件 SMTP；省略时按 **`smtp_address`**，否则 **`user@domain`**（`user` 无 `@` 时用 `domain` 拼接）。 |
| `--from-name` | 覆盖配置里的 `display_name`；中文等非 ASCII 默认会做 RFC2047 UTF-8 编码。 |
| `--from-name-plain-utf8` | 不对显示名做 RFC2047（少数服务器需试）。 |
| `--from-address-only` | 用纯 ASCII 邮箱作为显示名，减轻部分环境发件人乱码；建议配合完整 `--from`。 |

### 3.2 发送后复核「已发送」（默认开启）

| 参数 | 默认 | 说明 |
|------|------|------|
| `--verify-sent-wait` | `5` | 发送成功后等待的秒数，再在「已发送」中按主题查找副本（0～120）。 |
| `--no-verify-sent` | false | 不等待、不复核（脚本/CI 可加）。 |
| `--json` | false | 输出 JSON，含 `sent_verified`、`sent_verify_item_id` 等字段。 |

成功时典型输出：

- `发送成功。`
- 若 EWS 返回 ItemId 会多一行；
- 若复核命中：`已在「已发送」确认副本：… item_id=…`
- 若未命中：会提示可能延迟，请到网页「已发送」再确认。

### 3.3 示例

```text
mailcli send --config .\.mailcli.yml ^
  --from zhang.san@mail.example.com ^
  --to "a@b.com,c@d.com" ^
  --subject "通知" ^
  --text "正文内容"

mailcli send --config .\.mailcli.yml --to "x@y.z" --subject "无复核" --text "hi" --no-verify-sent --json
```

## 4. 查看单封邮件（`show`）

根据 **`search` 输出中的 ITEM_ID**（或 Outlook 等方式拿到的 EWS ItemId）拉取 **GetItem**：信头 + 正文；附件列出 **元数据**（名称/大小/类型），不下载二进制。

| 参数 | 说明 |
|------|------|
| `--item-id` | **必填**。EWS ItemId。 |
| `--change-key` | 可选；与搜索时一致可减少一次重试。 |
| `--format` | `text`（默认）、`html`、`json`。 |

示例：

```text
mailcli show --config .\.mailcli.yml --item-id "AAMkAG..." --format text

mailcli show --config .\.mailcli.yml --item-id "AAMkAG..." --format json
```

---

## 5. 排错提示

- 加 **`--verbose`** 查看 SOAP/HTTP 原文。
- 认证失败检查 `auth_type`、`domain`、密码与环境变量。
- `search` 在部分 on-prem 上对复杂条件可能 `ErrorInvalidRequest`，可尝试只加 `--subject` 或缩小条件。
- `show` 若报 **ChangeKey** 相关错误，可省略 `--change-key` 再试（工具会仅按 Id 重试）。

版本号：`mailcli --version`。
